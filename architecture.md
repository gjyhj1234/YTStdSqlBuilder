# 租户平台架构说明

## 概述

当前的租户平台架构设计旨在提供一个高效、可扩展和安全的多租户环境，以满足不同客户的需求。该架构涵盖了平台管理体系、租户生命周期管理、租户信息管理、资源管理、配置中心、套餐系统、订阅系统、计费与账单系统、API与集成平台、平台运营体系、日志与审计、通知系统、文件与存储以及技术基础设施等多个方面。

## 底层架构层
设计原则，轻量、高性能、尽可能使用Postgresql能力
目的：在较少的资源消耗下，提供高效、可靠的基础服务支持，满足多租户环境下的性能和安全需求。
原因：
1. 如果使用消息队列、分布式缓存等组件，可能会增加系统的复杂性和维护成本，尤其是在初期阶段。通过直接利用PostgreSQL的功能，可以简化架构，同时确保数据的一致性和安全性。
1. 避免系统在局域网系统安装时需要太多服务与服务器等资源的要求，降低部署和运维的门槛。

```
基础架构
│
├─ API Gateway
├─ 多租户中间件
├─ 权限中间件
├─ 限流中间件
├─ 配置中心
├─ 日志中心
├─ 监控中心
├─ 消息队列(可选)
├─ 对象存储
└─ 分布式缓存(可选)
```
