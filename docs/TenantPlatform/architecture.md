# 租户平台架构说明

## 概述

当前的租户平台架构设计旨在提供一个高效、易部署和安全的多租户环境，以满足不同客户的需求。

平台整体采用单体主程序架构：所有 WebAPI、权限判断、后台任务、配置管理与审计能力均运行在同一个主程序中，不拆分为微服务，也不引入分布式部署模式。

后续如有并发压力，只在入口层增加负载均衡节点，但业务服务本身仍保持单体部署。架构范围涵盖平台管理体系、租户生命周期管理、租户信息管理、资源管理、配置中心、套餐系统、订阅系统、计费与账单系统、API 与集成平台、平台运营体系、日志与审计、通知系统、文件与存储以及技术基础设施等多个方面。

## 底层架构层
设计原则：轻量、高性能、低内存占用、尽可能直接利用 PostgreSQL 能力。
目的：在较少的资源消耗下，提供高效、可靠的基础服务支持，满足多租户环境下的性能和安全需求，同时保证本地部署和局域网部署足够简单。
原因：
1. 租户平台后端不采用微服务拆分，所有服务能力统一收敛到一个主程序中，降低部署、排障与运维复杂度。
2. 即使后续发布阶段需要提升并发能力，也只在入口层增加负载均衡节点，不引入分布式业务节点、服务发现或分布式缓存依赖。
3. 权限判断、菜单可见性、角色授权结果、基础配置等高频读取数据统一采用进程内 Local Cache（本地缓存）方式实现，通过主动失效和定时刷新保证一致性。
4. 避免系统在局域网安装时要求额外的消息队列、缓存集群和服务治理组件，降低资源门槛。

```
基础架构
│
├─ 入口负载均衡(可选)
├─ 单体 WebAPI 主程序
│   ├─ 租户平台应用服务

│   ├─ 限流中间件
│   ├─ 进程内任务调度

├─ PostgreSQL

├─ 日志中心
└─ 监控中心
```
